# Dotfiles — Agent Instructions

## What This Is

Personal macOS dotfiles repo. Symlink-based management via `bin/install.sh` (no Chezmoi). Configures Zsh, Ghostty, tmux, Neovim, Oh My Posh, Claude Code, OpenCode, and Zed.

## Key Commands

```bash
./bin/install.sh workstation   # Full dev environment
./bin/install.sh minimal       # Shell + CLI tools only
./bin/install.sh links-only    # Just refresh symlinks (no package installs)
./bin/secrets.sh               # Restore secrets from Google Drive backup
```

Install script is idempotent. `links-only` is safe to re-run anytime.

## Architecture

- **Symlinks, not copies** — `bin/install.sh` symlinks dotfiles into `~/.*` and `~/.config/*/`. Existing files get backed up as `*.backup`.
- **Shell entry point**: `~/.zshrc` → sources `${DOTFILES}/shell/aliases.zsh` and `${DOTFILES}/shell/hooks.zsh`
- **Plugin manager**: Antidote (not Oh-My-Zsh directly). Plugins listed in `shell/plugins.txt`. Syntax highlighting must be last.
- **Package profiles**: `Brewfile` (base) → `Brewfile.workstation` or `Brewfile.media-server` layered on top. `mas` apps are skipped if not signed into App Store.
- **Secrets**: Stored in `~/.secrets` (symlinked from Google Drive), sourced by `.zshrc`. Never committed to repo.

## Conventions

- All shell scripts use `set -euo pipefail`
- Node versions via `fnm` (Rust-based); `.zshrc` runs `fnm env --use-on-cd` for auto-switching
- Oh My Posh prompt is skipped in Apple Terminal (only runs in Ghostty/tmux)
- `compinit` uses a 24-hour cache (`-C` flag) for faster shell startup
- `zcompdump` is background-compiled via `zcompile`

## tmux

- **Prefix is `Ctrl+A`** (not default `Ctrl+B`)
- **Sidebar plugin**: `tmux-window-sidebar` (Rust TUI in `tmux-window-sidebar/`) — replaces status bar with per-window agent cards
- **Agent hooks**: Claude Code hooks in `config/claude/settings.json` → `shell/tmux-hook.sh` → Rust binary. OpenCode plugin at `config/opencode/plugins/tmux-window-sidebar.ts`.
- **Agent icons**: ⚡ (running), ✓ (done), ❓ (waiting), ✕ (error) — set on tmux window names
- **Sound notifications**: `@agent-sound` (done, default: `Blow`), `@agent-ask-sound` (ask, default: random Navi sounds). Custom sounds in `sounds/` dir.
- **Window auto-rename**: `shell/hooks.zsh` renames windows based on cwd, preserving agent status icons. Only the direct pane shell renames (not agent subshells).
- **Dev layout**: `ccs` (Claude Code) or `ocs` (OpenCode) aliases start a 60/40 agent+editor split via `shell/dev-tmux-layout.zsh`

## tmux-window-sidebar (Rust Project)

Located at `tmux-window-sidebar/`. Build with `cargo build --release`. Binary installs to `~/.local/bin/tmux-window-sidebar`.

Key subcommands:
- `tmux-window-sidebar toggle` / `toggle-all` — show/hide sidebar
- `tmux-window-sidebar hook <agent> <event>` — receive agent events (called by hooks/plugins)
- `tmux-window-sidebar sync-status` — sync status bar with zoom state
- `tmux-window-sidebar focus-content` — move focus to content pane (skip sidebar)

Pane options data bus (`@pane_*` tmux options) connects hooks to the TUI. See `tmux-window-sidebar/PLAN.md` for full spec.

## OpenCode Config

- `config/opencode/opencode.json` — main config (theme, plugins, disabled agents)
- `config/opencode/AGENTS.md` — global agent instructions (provider/model info)
- `config/opencode/plugins/` — TypeScript plugins: `tmux-window-sidebar.ts`, `rtk.ts`
- `config/opencode/commands/` — slash commands (including `omo-models.md`)
- `config/opencode/skills/` — skill definitions
- `config/opencode/profiles/` — profile configs

## Claude Code Config

- `config/claude/settings.json` — hooks, permissions, plugins, model settings
- `config/claude/CLAUDE.md` — references `RTK.md` and `USER.md`
- Hooks bridge to tmux-window-sidebar via `shell/tmux-hook.sh`

## Git Config

- `pull.rebase = true` + `rebase.autoStash = true`
- `merge.conflictStyle = zdiff3`
- Delta as pager and diff filter
- `push.autoSetupRemote = true`

## Things Not to Break

- Don't modify `shell/plugins.zsh` — it's generated by Antidote (listed in `.gitignore`)
- Don't hardcode paths — use `${DOTFILES}` or `${HOME}` references
- Don't put secrets in the repo — they live in `~/.secrets`
- The `tmux-window-sidebar/target/` dir is gitignored — build artifacts only
- `config/opencode/plugins/tmux-window-sidebar.ts` is the agent status plugin (replaces the old `tmux-agent-status.ts`)
